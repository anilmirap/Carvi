# ROADMAP

## 1) Vision Path
Carvi will evolve from a workshop operations core into a full trust ecosystem connecting independent workshops and car owners.

## 2) Roadmap Principles
- Build trust infrastructure before growth features.
- Sequence by operational value, not superficial breadth.
- Stabilize workshop workflows before customer expansion.

## 3) Delivery Phases

### Phase 0 — Foundation (Current)
**Goal:** Align product and engineering direction.

Deliverables:
- Product positioning and scope documents
- Domain model language
- Delivery and implementation rules

### Phase 1 — Workshop Ops MVP (Priority)
**Goal:** Launch workshop-facing web app for daily operations.

Target capabilities:
- Workshop onboarding
- User roles
- Customer management
- Vehicle management
- Work order lifecycle
- Service + spare parts line items
- Quote approval workflow
- Service history
- Basic dashboard

Exit criteria:
- Pilot workshops can run core service operations digitally end-to-end
- Quote/approval and history records are auditable

### Phase 2 — MVP Hardening & Operational Reliability
**Goal:** Improve quality, reliability, and usability for scale-up.

Focus areas:
- Permission model hardening
- Workflow edge-case handling
- Reporting refinements
- Performance and operational monitoring
- Data quality tooling

### Phase 3 — Customer Experience Layer
**Goal:** Introduce customer-facing mobile/web surfaces.

Target capabilities:
- Customer account access
- Vehicle service timeline visibility
- Quote review/approval interfaces
- Service status transparency

### Phase 4 — Ecosystem Expansion
**Goal:** Extend Carvi into a broader service network platform.

Potential expansions:
- Warranty process management
- Supplier/procurement integrations
- Multi-branch operations
- Advanced analytics and benchmarking
- Marketplace trust signals and discovery enhancements

## 4) Near-Term Milestones (Indicative)
1. Documentation baseline approved
2. MVP architecture and implementation plan approved
3. MVP alpha (internal)
4. MVP beta (pilot workshops)
5. MVP v1 launch

## 5) Prioritization Framework
For backlog decisions, prioritize items that maximize:
1. Trust and traceability
2. Operational throughput for workshops
3. Data integrity and workflow correctness
4. User clarity and adoption

## 6) Risks to Watch
- Scope creep before core workflow stability
- Overbuilding secondary features before MVP completion
- Inconsistent domain naming across teams
- Weak approval/audit design reducing trust promise

## 7) Definition of Strategic Success
Carvi succeeds when independent workshops can operate with enterprise-level process quality, and car owners experience transparent, reliable, professional service as the new normal.
